# 🤝 Contribuir al Proyecto

¡Gracias por querer contribuir! Este documento explica cómo hacerlo.

## 📋 Cómo Contribuir

### 1. **Reportar Bugs**

Si encuentras un error:

1. Ve a [Issues](https://github.com/tu-usuario/Aprendizaje-Programacion/issues)
2. Click en "New Issue"
3. Describe el problema detalladamente

**Incluye:**

- ¿Qué paso?
- ¿Qué esperabas que pasara?
- ¿Cómo reproducirlo?
- Tu sistema operativo y versión

### 2. **Sugerir Mejoras**

¿Tienes una idea?

1. Abre un [Issue](https://github.com/tu-usuario/Aprendizaje-Programacion/issues)
2. Describe tu idea
3. Explica por qué sería útil

### 3. **Enviar Código (Pull Request)**

#### Paso 1: Fork el Repositorio

```bash
Click en "Fork" en la página del repo
```

#### Paso 2: Clona tu Fork

```bash
git clone https://github.com/TU-USUARIO/Aprendizaje-Programacion.git
cd Aprendizaje-Programacion
```

#### Paso 3: Crea una Rama

```bash
git checkout -b mejora/nombre-de-tu-mejora
```

#### Paso 4: Haz los Cambios

- Edita archivos
- Agrega ejemplos
- Mejora documentación
- Etc.

#### Paso 5: Commit tus Cambios

```bash
git add .
git commit -m "Agrego nueva lección sobre X"
```

#### Paso 6: Push a tu Fork

```bash
git push origin mejora/nombre-de-tu-mejora
```

#### Paso 7: Abre un Pull Request

1. Ve a GitHub
2. Verás un botón "Compare & Pull Request"
3. Describe tus cambios
4. Click en "Create Pull Request"

## 📝 Estándares

### Código Python

```python
# Formato PEP 8
# Líneas máximo 79 caracteres
# Nombres descriptivos
# Comentarios claros

def mi_funcion():
    """Descripción de la función"""
    pass
```

### Documentación Markdown

```markdown
# Título Principal

## Subtítulo

**Bold** _Italic_
`código inline`

\`\`\`python
código de bloque
\`\`\`
```

### Estructura de Ejemplos

```
archivo.py
├── Docstring del módulo
├── Imports
├── Funciones con docstrings
├── if __name__ == "__main__":
└── Código principal
```
