## 🗺️ Tu Ruta Exacta (Semana 1-12)

### 🔵 SEMANA 1-3: Pseint (Sin Programación Real)

**¿Qué es?** Un programa donde escribes lógica en español, sin "código complicado"

**Por qué?** Aprendes a PENSAR como programador sin sintaxis

**Materiales:**

- Descargas: https://pseint.sourceforge.net/
- Lee: [01-Pseint/README.md](../01-Pseint/README.md)
- Tiempo: 2-3 semanas

**Temas que aprenderás:**

1. Variables (dónde guardas datos)
2. Tipos de datos (números, texto, verdadero/falso)
3. Operadores (suma, resta, comparación)
4. Condicionales (si esto, entonces eso)
5. Bucles (repetir acciones)
6. Funciones (código reutilizable)

**Cómo estudiar:**

```
CADA DÍA:
1. Lee 1 tema (15 min)
2. Entiende el concepto (15 min)
3. Escribe el ejemplo en Pseint (20 min)
4. Haz 1 ejercicio (20 min)
5. ¿Dudas? Pregunta a Claude (con QUICK_START_CLAUDE.md)
```

**¿Fácil?** Sí. No es programación "de verdad" todavía.

### 🟢 SEMANA 4-9: Python (Programación Real)

**¿Qué es?** Un lenguaje de programación fácil y poderoso

**Por qué?** Ahora implementas tus ideas en un lenguaje real

**Materiales:**

- Descargas: https://www.python.org/ (Python 3.11+)
- Lee: [02-Python/README.md](../02-Python/README.md)
- Código: [02-Python/ejemplos/](../02-Python/ejemplos/)
- Tiempo: 4-6 semanas

**Temas que aprenderás:**

1. Sintaxis Python (cómo escribir código)
2. Tipos de datos (igual que Pseint, pero en Python)
3. Listas y diccionarios (guardar varios datos)
4. Funciones (código reutilizable)
5. Manejo de archivos (leer/escribir en computadora)
6. Excepciones (qué hacer cuando falla)

**Cómo estudiar:**

```
CADA DÍA:
1. Lee 1 tema (20 min)
2. Ve ejemplo en Python (10 min)
3. Ejecuta: python ejemplo.py (5 min)
4. Modifica el ejemplo (20 min)
5. Haz 1 ejercicio (20 min)
6. ¿Dudas? Pregunta a Claude
```

**Ejemplos que puedes ejecutar:**

- `01-conversor-temperatura.py` - Convertir temperaturas
- `02-lista-tareas.py` - Tu primer "app"
- `03-juego-adivinanza.py` - Juego interactivo
- `04-contador-palabras.py` - Análisis de textos
- `05-gestor-contactos.py` - Base de datos simple

**¿Difícil?** Un poco más, pero tiene la base de Pseint.

### 🟡 SEMANA 10-13: Objetos (Programación Avanzada)

**¿Qué es?** Forma más profesional de organizar código

**Por qué?** Proyectos grandes necesitan esta estructura

**Materiales:**

- Lee: [03-OOP/README.md](../03-OOP/README.md)
- Tiempo: 3-4 semanas

**Temas que aprenderás:**

1. Clases y objetos
2. Atributos (datos del objeto)
3. Métodos (acciones del objeto)
4. Herencia (reutilizar clases)
5. Encapsulación (proteger datos)

**Cómo estudiar:**

```
Similar a Python, pero:
1. Entiendes conceptos (más teóricos)
2. Ves ejemplos de código
3. Escribes pequeñas clases
4. Haces ejercicios progresivos
```

**¿Difícil?** Sí, es un salto. Pero tendrás Claude.

## ❌ ERRORES A EVITAR

### Error 1: Solo leer sin escribir código

❌ **Mal:**

```
Leo: "Las variables son..."
Entiendo la idea
Sigo al siguiente tema
```

✅ **Bien:**

```
Leo: "Las variables son..."
Escribo en Pseint/Python
Ejecuto el código
Modifico para experimentar
```

**Por qué:** La programación se aprende HACIENDO, no viendo.

### Error 2: No preguntar cuando no entiendes

❌ **Mal:**

```
No entiendo variables
Intento pasar al siguiente tema
Me pierdo más adelante
```

✅ **Bien:**

```
No entiendo variables
Pregunto a Claude
Leo 3 explicaciones diferentes
Entiendo
Continúo
```

**Por qué:** Cada concepto se necesita para el siguiente.

### Error 3: Saltar Pseint

❌ **Mal:**

```
"Pseint es para niños, voy directo a Python"
→ Me pierdo en sintaxis
→ Abandono
```

✅ **Bien:**

```
Hago Pseint (2-3 semanas)
Aprendo lógica pura
Python es fácil después
```

**Por qué:** Pseint enseña lógica. Python enseña sintaxis. Necesitas ambas.

### Error 4: Copiar código sin entender

❌ **Mal:**

```
CLAUDE: [Da código]
TÚ: Copias
Pegas
Se rompe
No sabes por qué
```

✅ **Bien:**

```
CLAUDE: [Da código]
TÚ: 1. Lees línea por línea
    2. Ejecutas
    3. Modificas
    4. Preguntas por qué
    5. Entiende
```

**Por qué:** Si no entiendes, cuando algo falla estarás perdido.
